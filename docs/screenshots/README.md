# Screenshots — capture checklist

Drop the PNGs in this folder using the exact filenames below, then **uncomment
the screenshot table** in the root `README.md` (`### 📸 Screenshots`).

The Obsidian Community Plugins submission strongly benefits from at least one
desktop + one mobile shot. Aim for a clean demo vault (a few real-looking notes,
no personal data, no visible API keys).

## Shots needed

| Filename | What to capture | Notes |
|---|---|---|
| `desktop-chat.png` | A finished chat in the right sidebar, mid-conversation | Show streaming UI + a code block with copy button |
| `mobile-drawer.png` | The mobile drawer with the composer | Edge-to-edge composer; ideally on a real phone or mobile emulator |
| `vault-qa.png` | A Vault Q&A answer **with `[[wikilink]]` citations** | The citations are a headline feature — make them visible |
| `agent-diff.png` | The agent's diff-approval confirmation dialog | Show the red/green diff preview before applying |
| `usage-dashboard.png` | The Usage dashboard with the 30-day heatmap | Pick a vault with enough history to look populated |
| `starter.png` | The starter screen (provider → model → mode → effort) | Shows the segmented controls + effort slider |
| `sidebar-nav.png` | The side nav (sections + "show more") | Open via the avatar; shows Conversas/Mídia/Estatísticas + expand |
| `image-gen.png` | In-chat image generation (modal + result) | `+` → Create image, or the agent calling generate_image |
| `onboarding.png` | First-run onboarding (zero keys) | Fresh install / no API key set |
| `plans.png` | Plans screen (Free vs Pro + license) | LockedScreen "See plans", or nav while free |

## Guidelines

- **Format:** PNG, light **and**/or dark theme (dark usually reads best for AXXA).
- **Width:** ~1200–1600px for desktop; native device width for mobile.
- **Privacy:** scrub any real note titles, paths, or keys. Use a throwaway demo vault.
- **Crop:** tight to the plugin panel — avoid showing the whole desktop.

After adding them, also refresh the GIF mentioned in the roadmap (a 20–30s
loop: open → ask vault → agent edits a note → approve diff).
